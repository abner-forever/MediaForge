# 用户系统使用指南

## 概述

MediaForge 现已支持用户系统，用户可以通过邮箱注册账号，实现：
- 跨设备同步积分、设置、文章草稿等数据
- 多设备登录管理
- 云端数据备份

## 用户注册

### 1. 打开注册页面
- 点击侧边栏底部的"登录"按钮
- 在登录页面点击"没有账号？立即注册"

### 2. 填写注册信息
- **邮箱地址**：用于登录和接收验证码
- **昵称**：显示名称，1-20个字符
- **密码**：至少6位
- **确认密码**：再次输入密码
- **邮箱验证码**：点击"发送验证码"按钮获取

### 3. 完成注册
- 填写所有信息后点击"注册"按钮
- 注册成功后自动登录并跳转到首页

## 用户登录

### 1. 打开登录页面
- 点击侧边栏底部的"登录"按钮

### 2. 输入账号信息
- **邮箱地址**：注册时使用的邮箱
- **密码**：账号密码

### 3. 登录
- 点击"登录"按钮
- 登录成功后自动跳转到首页

## 用户中心

### 访问用户中心
- 登录后，点击侧边栏底部的用户名或"用户中心"

### 功能说明

#### 1. 个人信息
- 查看用户ID、注册时间、最后登录时间
- 修改昵称（点击昵称旁的编辑图标）

#### 2. 绑定设备
- 查看已绑定的设备列表
- 解绑不再使用的设备

#### 3. 退出登录
- 点击"退出登录"按钮

## 数据同步

### 自动同步
- 用户登录后，积分、设置等数据会自动同步到云端
- 换设备登录时，自动从云端恢复数据

### 手动同步
- 在"积分中心"页面的"云同步"卡片中
- 点击"立即同步"按钮手动触发同步

## 开发者部署

### 1. 部署云同步服务
```bash
cd cloud_sync
pip install -r requirements.txt
python server.py
```

### 2. 配置环境变量
```bash
export SYNC_SECRET="your-secret-key"
export SYNC_DATA_DIR="./data"
export SYNC_PORT=8080
```

### 3. 启动服务
```bash
python server.py
```

### 4. 配置客户端
在应用中配置同步服务地址和密钥。

## 技术实现

### 后端
- **用户服务**：`services/user.py`
- **用户路由**：`desktop/routers/user.py`
- **数据存储**：`data/users/` 目录下的 JSON 文件

### 前端
- **类型定义**：`desktop/web/src/types/user.ts`
- **API 模块**：`desktop/web/src/api/user.ts`
- **状态管理**：`desktop/web/src/stores/userSlice.ts`
- **登录页面**：`desktop/web/src/pages/Auth/index.tsx`
- **用户中心**：`desktop/web/src/pages/UserCenter/index.tsx`

### 认证机制
- JWT Token 认证
- Token 存储在 localStorage
- 自动刷新和过期处理

## 安全说明

1. **密码安全**：密码使用 SHA256 + 盐值哈希存储
2. **邮箱验证**：注册时需要邮箱验证码
3. **Token 安全**：JWT Token 有效期 30 天
4. **设备绑定**：支持多设备管理，可随时解绑

## 常见问题

### Q: 忘记密码怎么办？
A: 目前暂不支持密码重置功能，后续版本会添加。

### Q: 可以修改邮箱吗？
A: 目前暂不支持修改邮箱，后续版本会添加。

### Q: 如何解绑设备？
A: 在用户中心的"绑定设备"部分，点击设备旁的"解绑"按钮。

### Q: 数据会丢失吗？
A: 登录状态下，数据会自动同步到云端。退出登录后，本地数据仍保留。
