## 一、功能概述

### 1.1 业务背景

MediaForge 是一个自动化微信公众号内容发布系统。为了实现商业化变现，计划引入积分系统，用户通过观看广告、签到、邀请等方式获取积分，每次发布文章需要消耗积分。

### 1.2 核心目标

1. **变现模式** - 通过广告观看实现用户变现
2. **用户激励** - 提供多种积分获取途径，提高用户活跃度
3. **成本控制** - 通过积分消耗控制发布频率
4. **用户体验** - 积分系统不影响核心工作流程

### 1.3 目标用户

- 个人公众号运营者
- 内容创作者
- 多账号矩阵运营者

### 3.2 积分系统

#### 3.2.1 积分获取方式

**1. 看广告赚积分**
- 每观看1个完整广告 = 10积分
- 每日上限：100积分（10个广告）
- 广告时长：15-30秒

**2. 每日签到**
- 连续签到递增奖励：
  - 第1天：5积分
  - 第2天：10积分
  - 第3天：15积分
  - 第4天：20积分
  - 第5天：25积分
  - 第6天：30积分
  - 第7天：50积分（额外奖励）
- 连续签到7天后重置

**3. 邀请好友**
- 邀请1位好友注册：100积分
- 好友首次发布：额外50积分
- 无上限

**4. 任务系统**
- 首次发布文章：50积分
- 发布10篇文章：200积分
- 使用AI生成文章：20积分
- 完善个人资料：30积分

**5. 购买积分**
- 1元 = 100积分
- 最低购买：100积分（1元）
- 支持微信支付/支付宝

#### 3.2.2 积分消耗

**发布文章消耗：**
- 普通发布：10积分/篇
- AI生成文章：额外20积分/篇
- 定时发布：额外5积分/篇
- 多账号发布：每个额外账号5积分

**其他消耗：**
- 优先队列：5积分/次
- 专属模板：10积分/次
- 高清图片下载：2积分/张

### 3.3 广告系统

#### 3.3.1 广告类型

**1. 激励视频广告**
- 时长：15-30秒
- 用户主动选择观看
- 观看完整视频获得积分
- 每日限制次数

**2. 插屏广告**
- 在特定操作后展示
- 用户可选择关闭
- 不强制观看

**3. Banner广告**
- 在页面底部展示
- 不影响主要操作
- 点击可获得额外积分

#### 3.3.2 广告平台选择

**推荐方案：穿山甲（字节跳动）**
- 优势：国内主流、收益稳定、SDK完善
- 劣势：主要面向移动端，桌面应用需要适配

**备选方案：**
1. **广点通（腾讯）** - 腾讯广告平台
2. **百青藤（百度）** - 百度广告平台
3. **Google AdMob** - 国际平台，需要VPN

**技术实现：**
- 使用PyWebView嵌入广告页面
- 通过JavaScript Bridge与原生通信
- 服务端验证广告观看状态

#### 3.3.3 广告验证流程

```
用户点击"看广告"
    ↓
前端请求广告配置
    ↓
后端返回广告ID和签名
    ↓
前端加载广告SDK
    ↓
用户观看广告
    ↓
广告SDK回调（客户端）
    ↓
前端发送验证请求到后端
    ↓
后端验证签名和观看状态
    ↓
增加积分并返回结果
```

## 五、API 设计

### 5.1 用户相关 API

#### 注册/登录
```
POST /api/auth/register
POST /api/auth/login
POST /api/auth/logout
GET  /api/auth/me
```

#### 用户信息
```
GET    /api/user/profile
PUT    /api/user/profile
GET    /api/user/points
GET    /api/user/points/log
```

### 5.2 积分相关 API

#### 积分操作
```
POST /api/points/earn/ad        # 观看广告获得积分
POST /api/points/earn/checkin   # 签到获得积分
POST /api/points/earn/invite    # 邀请获得积分
POST /api/points/earn/task      # 任务获得积分
POST /api/points/spend/publish  # 发布消耗积分
```

#### 积分查询
```
GET /api/points/balance         # 查询余额
GET /api/points/log             # 查询记录
GET /api/points/daily-limit     # 查询每日限制
```

### 5.3 广告相关 API

#### 广告配置
```
GET /api/ads/config             # 获取广告配置
GET /api/ads/rewarded           # 获取激励视频广告
POST /api/ads/verify            # 验证广告观看
```

### 5.4 支付相关 API

#### 订单管理
```
POST /api/orders/create         # 创建订单
GET  /api/orders/{order_id}     # 查询订单
POST /api/orders/{order_id}/pay # 发起支付
POST /api/orders/callback       # 支付回调
```

## 七、安全设计

### 7.1 防作弊机制

1. **广告验证**
   - 客户端签名验证
   - 服务端二次验证
   - 设备指纹绑定

2. **签到防刷**
   - IP限制
   - 设备限制
   - 时间间隔检查

3. **邀请防刷**
   - 邀请码有效期
   - 新用户验证
   - 异常检测

### 7.2 数据安全

1. **用户数据加密**
   - 手机号脱敏存储
   - 密码加密存储
   - 敏感信息加密

2. **支付安全**
   - HTTPS传输
   - 签名验证
   - 订单防重

## 九、风险评估

### 9.1 技术风险

| 风险 | 影响 | 概率 | 应对措施 |
|------|------|------|---------|
| 广告平台SDK不支持桌面 | 高 | 中 | 使用WebView嵌入，或选择支持桌面的广告平台 |
| 支付接口审核严格 | 中 | 中 | 提前准备资质材料，选择合适的支付渠道 |
| 数据存储性能问题 | 中 | 低 | 优化JSON结构，考虑引入SQLite |

### 9.2 业务风险

| 风险 | 影响 | 概率 | 应对措施 |
|------|------|------|---------|
| 用户接受度低 | 高 | 中 | 提供免费积分，优化用户体验 |
| 广告收入低于预期 | 中 | 中 | 多种变现方式并行，优化广告策略 |
| 竞品模仿 | 低 | 高 | 持续创新，建立用户粘性 |
